# @humonics/sdk

The official TypeScript SDK for the Humonics protocol. Verify content authenticity, issue certificates, and interact with the Humonics Soroban smart contracts — in one minimal API.

Built for publishers, hiring platforms, AI dataset curators, and legal platforms that need cryptographic proof of content origin.

---

## Installation

```bash
npm install @humonics/sdk
```

---

## Quick start

```typescript
import { HumonicsClient } from '@humonics/sdk';

const client = new HumonicsClient({ network: 'mainnet' });

const result = await client.verify('a3f1...'); // SHA-256 hex of your content

if (result.certified) {
  console.log('Certified by:', result.certificate?.did);
  console.log('Issued at:', result.certificate?.issuedAt);
} else {
  console.log('Not certified');
}
```

## API

### `new HumonicsClient(config)`

```typescript
const client = new HumonicsClient({
  network: 'mainnet' | 'testnet',
  rpcUrl?: string,       // Override default Soroban RPC endpoint
  horizonUrl?: string,   // Override default Horizon endpoint
  debug?: (msg: string) => void,  // Optional debug logger — never logs internally
});
```

---

### `client.verify(contentHash)`

Checks whether a piece of content has a valid certificate on-chain.

```typescript
const result: VerificationResult = await client.verify(contentHash: string);
```

- `contentHash` — SHA-256 hex string (64 chars) of the content
- Returns `{ certified: false }` for unknown content — **never throws** for uncertified content
- Returns `{ certified: true, certificate, revoked: true }` if the certificate exists but was revoked

```typescript
interface VerificationResult {
  certified: boolean;
  certificate?: Certificate;
  revoked?: boolean;
}
```

---

### `client.batchVerify(contentHashes)`

Verify up to 100 content hashes in a single call. Runs in parallel.

```typescript
const results: VerificationResult[] = await client.batchVerify(contentHashes: string[]);
```

---

### `client.issue(input)`

Issue a certificate for a piece of content. Requires a verified creator with a Stellar keypair and a valid zkProof.

```typescript
const cert: Certificate = await client.issue({
  contentHash: string,     // SHA-256 hex of the content
  zkProof: ZKProof,        // Generated by the zk-circuits package
  did: string,             // did:stellar:G...
  contentType: ContentType, // 'text' | 'code' | 'art' | 'audio' | 'video'
  keypair: Keypair,        // Stellar keypair — used to sign the transaction only
});
```

The zkProof is validated locally before any network call is made. The keypair is used only to sign the Stellar transaction and is never stored or forwarded.

---

### `client.revoke(certId, keypair)`

Revoke a previously issued certificate.

```typescript
await client.revoke(certId: string, keypair: Keypair);
```

Only the original issuer (matching keypair) can revoke a certificate.

## Return types

```typescript
interface Certificate {
  id: string;
  contentHash: string;
  did: string;
  contentType: 'text' | 'code' | 'art' | 'audio' | 'video';
  issuedAt: number;   // Unix timestamp
  txHash: string;     // Stellar transaction hash
  revoked: boolean;
}

interface ZKProof {
  pi_a: string[];
  pi_b: string[][];
  pi_c: string[];
  protocol: string;   // 'groth16'
  curve: string;      // 'bn128'
}
```

## Error handling

All errors are wrapped in `HumonicsError` — raw Stellar/Soroban errors never reach your code.

```typescript
import { HumonicsClient, HumonicsError } from '@humonics/sdk';

try {
  const cert = await client.issue({ ... });
} catch (err) {
  if (err instanceof HumonicsError) {
    switch (err.code) {
      case 'INVALID_PROOF':     // zkProof failed local validation
      case 'INVALID_INPUT':     // Bad contentHash, DID, etc.
      case 'CONTRACT_ERROR':    // Soroban contract rejected the call
      case 'NETWORK_ERROR':     // RPC unreachable
      case 'CERTIFICATE_REVOKED':
      case 'CONTENT_NOT_CERTIFIED':
    }
  }
}
```

| Code | When |
|---|---|
| `CONTENT_NOT_CERTIFIED` | `verify()` found no certificate (returned, not thrown) |
| `CERTIFICATE_REVOKED` | Certificate exists but is revoked |
| `INVALID_PROOF` | zkProof failed local validation before submission |
| `INVALID_INPUT` | Bad contentHash format, invalid DID, missing fields |
| `CONTRACT_ERROR` | Soroban contract returned an error |
| `NETWORK_ERROR` | RPC or Horizon unreachable |

## Examples

### Verify content

```typescript
import { HumonicsClient } from '@humonics/sdk';
import { createHash } from 'crypto';

const client = new HumonicsClient({ network: 'mainnet' });

const contentHash = createHash('sha256').update(myContent).digest('hex');
const result = await client.verify(contentHash);

if (result.certified && !result.revoked) {
  // Safe to trust
}
```

### Issue a certificate

```typescript
import { HumonicsClient } from '@humonics/sdk';
import { Keypair } from '@stellar/stellar-sdk';

const client = new HumonicsClient({ network: 'testnet' });
const keypair = Keypair.fromSecret(process.env.STELLAR_SECRET_KEY);

const cert = await client.issue({
  contentHash: '...',
  zkProof: { pi_a: [...], pi_b: [...], pi_c: [...], protocol: 'groth16', curve: 'bn128' },
  did: `did:stellar:${keypair.publicKey()}`,
  contentType: 'text',
  keypair,
});

console.log('Certificate ID:', cert.id);
```
